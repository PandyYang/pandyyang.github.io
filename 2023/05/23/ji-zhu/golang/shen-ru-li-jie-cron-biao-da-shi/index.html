深入理解Cron表达式
Golang

Cron即定时任务。这个词，来自于希腊语chronos，原意是时间。

cron是类Unix操作系统下的基于时间的任务管理系统。crontab（任务时间表）文件存储的指令被crond守护进程激活，守护进程在后台运行，并每一分钟检查是否有定期的作业需要执行，这类作业一般被称为cron jobs。

这篇文章，会通过Linux操作系统上的cron和Golang的三方库robfig/cron，来系统分析学习此工具的用法。

注意时区问题，任何cron都是以运行机器的当前时区进行处理的！所以必须要矫正时区！

语法

注意：

Linux的crontab的最小执行单位是分钟，没有直接实现单位秒的运行。

理解 Cron 表达式的关键是理解其语法和各个字段的含义。Cron 表达式由空格分隔的 6 个字段组成，表示时间的不同部分。以下是 Cron 表达式的各个字段及其含义：

分钟（0-59） 小时（0-23） 日期（1-31） 月份（1-12） 星期几（0-6，星期一至星期日）

Field name   | Mandatory? | Allowed values  | Allowed special characters
----------   | ---------- | --------------  | --------------------------
Minutes      | Yes        | 0-59            | * / , -
Hours        | Yes        | 0-23            | * / , -
Day of month | Yes        | 1-31            | * / , - ?
Month        | Yes        | 1-12 or JAN-DEC | * / , -
Day of week  | Yes        | 0-6 or SUN-SAT  | * / , - ?

每个字段可以使用数字、逗号、连字符、星号、正斜杠来制定不同的取值和范围，例如：

  • *：使用*的域可以匹配任何值，例如将月份域（第 4 个）设置为*，表示每个月；
  • /：用来指定范围的步长，例如将小时域（第 2 个）设置为3-59/15表示第 3 分钟触发，以后每隔 15 分钟触发一次，因此第 2 次触发为第 18 分钟，第 3 次为 33 分钟。。。直到分钟大于 59；
  • ,：用来列举一些离散的值和多个范围，例如将周历的域（第 5 个）设置为MON,WED,FRI表示周一、三和五；
  • -：用来表示范围，例如将小时的域（第 1 个）设置为9-17表示上午 9 点到下午 17 点（包括 9 和 17）；
  • ?：只能用在月历和周历的域中，用来代替*，表示每月/周的任意一天。

此外，Cron表达式还支持一些特殊字符：

  • ?：用于表示该字段不指定具体值，通常用于日期和星期几字段。例如，0 0 ? * * * 表示每天的任意时间执行一次任务。
  • L：用于表示月份中的最后一天或星期几中的最后一个实例。例如，0 0 0 L * ? 表示每个月的最后一天午夜执行一次任务。
  • W：用于表示最接近指定日期的工作日（周一至周五）。例如，0 0 12 1W * ? 表示每个月的最接近第一个工作日中午 12 点执行一次任务。
  • #：用于表示月份中的第几个星期几。例如，0 0 12 ? * 2#1 表示每个月的第一个星期一中午 12 点执行一次任务。

示例

看了语法，是不是感觉懵了？光学不练假把式。

  1. 每晚两点

    0 2 * * *

    其实学到这里我的问题就解决了，每晚两点跑一次程序升级任务。

    next at 2023-05-24 02:00:00
then at 2023-05-25 02:00:00
then at 2023-05-26 02:00:00
then at 2023-05-27 02:00:00
then at 2023-05-28 02:00:00

  2. “At 00:05 in August.”

    5 0 * 8 *

    在8月的每天的00 : 05。

    next at 2023-08-01 00:05:00
then at 2023-08-02 00:05:00
then at 2023-08-03 00:05:00
then at 2023-08-04 00:05:00
then at 2023-08-05 00:05:00

    注意：只在每年8月的每天凌晨5分的时候执行。

  3. “At minute 5 in August.”

    在8月的每小时的第五分钟。

    5 * * 8 *

    next at 2023-08-01 00:05:00
then at 2023-08-01 01:05:00
then at 2023-08-01 02:05:00
then at 2023-08-01 03:05:00
then at 2023-08-01 04:05:00

  4. “At minute 5 on Sunday in August.”

    5 * * 8 0

    next at 2023-08-06 00:05:00
then at 2023-08-06 01:05:00
then at 2023-08-06 02:05:00
then at 2023-08-06 03:05:00
then at 2023-08-06 04:05:00

  5. “At minute 5 on day-of-month 1 and on Sunday in August.”

    在 8 月的第 1 天和星期日的第 5 分钟。

    5 * 1 8 0

    next at 2023-08-01 00:05:00
then at 2023-08-01 01:05:00
then at 2023-08-01 02:05:00
then at 2023-08-01 03:05:00
then at 2023-08-01 04:05:00

  6. “At minute 5 on every day-of-month from 1 through 22 and on Sunday in August.”

    每月 1 日到 22 日以及 8 月的星期日的第 5 分钟。

    5 * 1-22 8 0

    next at 2023-08-01 00:05:00
then at 2023-08-01 01:05:00
then at 2023-08-01 02:05:00
then at 2023-08-01 03:05:00
then at 2023-08-01 04:05:00

  7. “At minute 5 past every 2nd hour on every day-of-month from 1 through 22 and on Sunday in August.”

    每月 1 日至 22 日以及 8 月的星期日，每 2 小时第 5 分钟。

    5 */2 1-22 8 0

    next at 2023-08-01 00:05:00
then at 2023-08-01 02:05:00
then at 2023-08-01 04:05:00
then at 2023-08-01 06:05:00
then at 2023-08-01 08:05:00

预定义时间规则

为了方便使用，cron预定义了一些时间规则：

  • @yearly：也可以写作@annually，表示每年第一天的 0 点。等价于0 0 1 1 *；
  • @monthly：表示每月第一天的 0 点。等价于0 0 1 * *；
  • @weekly：表示每周第一天的 0 点，注意第一天为周日，即周六结束，周日开始的那个 0 点。等价于0 0 * * 0；
  • @daily：也可以写作@midnight，表示每天 0 点。等价于0 0 * * *；
  • @hourly：表示每小时的开始。等价于0 * * * *。

Linux

Linux crontab 是用来定期执行程序的命令。

当安装完成操作系统之后，默认便会启动此任务调度命令。

crond 命令每分钟会定期检查是否有要执行的工作，如果有要执行的工作便会自动执行该工作。

注意：新创建的 cron 任务，不会马上执行，至少要过 2 分钟后才可以，当然你可以重启 cron 来马上执行。

而 linux 任务调度的工作主要分为以下两类：

  • 1、系统执行的工作：系统周期性所要执行的工作，如备份系统数据、清理缓存
  • 2、个人执行的工作：某个用户定期要做的工作，例如每隔 10 分钟检查邮件服务器是否有新信，这些工作可由每个用户自行设置

语法：

crontab [ -u user ] file

crontab [ -u user ] { -l | -r | -e }

参数说明：

  • -e : 执行文字编辑器来设定时程表，内定的文字编辑器是 VI，如果你想用别的文字编辑器，则请先设定 VISUAL 环境变数来指定使用那个文字编辑器(比如说 setenv VISUAL joe)
  • -r : 删除目前的时程表
  • -l : 列出目前的时程表

具体用法：

  1. 查看当前定时任务。

    crontab -l

  2. 编辑当前的定时任务

    crontab -e

具体示例：

0 */2 * * * /sbin/service httpd restart  意思是每两个小时重启一次apache 

50 7 * * * /sbin/service sshd start  意思是每天7：50开启ssh服务 

50 22 * * * /sbin/service sshd stop  意思是每天22：50关闭ssh服务 

0 0 1,15 * * fsck /home  每月1号和15号检查/home 磁盘 

1 * * * * /home/bruce/backup  每小时的第一分执行 /home/bruce/backup这个文件 

00 03 * * 1-5 find /home "*.xxx" -mtime +4 -exec rm {} \;  每周一至周五3点钟，在目录/home中，查找文件名为*.xxx的文件，并删除4天前的文件。

30 6 */10 * * ls  意思是每月的1、11、21、31日是的6：30执行一次ls命令

如果你要在Linux中实现秒的方式，可以使用：

*/1 * * * * /root/python.sh
*/1 * * * * sleep 15; /root/python.sh
*/1 * * * * sleep 30; /root/python.sh
*/1 * * * * sleep 45; /root/python.sh

Golang

看完了上面的语法，基本上熟悉了Corn的使用。在程序中体会下如何使用。

下载对应的库。

go get github.com/robfig/cron/v3@v3.0.0

调用者可以注册 Funcs 以在给定的时间表上调用。 Cron 将在它们自己的 goroutine 中运行它们。

c := cron.New()
c.AddFunc("0 30 * * * *", func() { fmt.Println("Every hour on the half hour") })
c.AddFunc("@hourly",      func() { fmt.Println("Every hour") })
c.AddFunc("@every 1h30m", func() { fmt.Println("Every hour thirty") })
c.Start()
..
// Funcs are invoked in their own goroutine, asynchronously.
...
// Funcs may also be added to a running Cron
c.AddFunc("@daily", func() { fmt.Println("Every day") })
..
// Inspect the cron job entries' next and previous run times.
inspect(c.Entries())
..
c.Stop()  // Stop the scheduler (does not stop any jobs already running).

表达式：

注意上面的语法没有Seconds，Golang这个库自己封装了秒级实现。

Field name   | Mandatory? | Allowed values  | Allowed special characters
----------   | ---------- | --------------  | --------------------------
Seconds      | Yes        | 0-59            | * / , -
Minutes      | Yes        | 0-59            | * / , -
Hours        | Yes        | 0-23            | * / , -
Day of month | Yes        | 1-31            | * / , - ?
Month        | Yes        | 1-12 or JAN-DEC | * / , -
Day of week  | Yes        | 0-6 or SUN-SAT  | * / , - ?

注意：月份和星期几字段值不区分大小写。 “SUN”、“Sun”、“sun”等同接受。

此库的预定义时间。

Entry                  | Description                                | Equivalent To
-----                  | -----------                                | -------------
@yearly (or @annually) | Run once a year, midnight, Jan. 1st        | 0 0 0 1 1 *
@monthly               | Run once a month, midnight, first of month | 0 0 0 1 * *
@weekly                | Run once a week, midnight between Sat/Sun  | 0 0 0 * * 0
@daily (or @midnight)  | Run once a day, midnight                   | 0 0 0 * * *
@hourly                | Run once an hour, beginning of hour        | 0 0 * * * *

一个示例。

本示例演示了当http服务启动时，通过一个go协程去拉起一个定时任务，在每天的凌晨两点对数据库进行更新操作。

package main

import (
    "fmt"
    "log"
    "net/http"
    "time"
)

func main() {
    go startScheduledTask()

    http.HandleFunc("/", handler)

    log.Fatal(http.ListenAndServe(":8080", nil))
}

func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprint(w, "Hello, World!")
}

func startScheduledTask() {
    c := cron.New()
    // 每天凌晨2点执行的cron表达式
    c.AddFunc("0 0 2 * * *", func ()  {
        updateDatabase()
    })

    c.Start()
}

func updateDatabase() {
    // 实现后台更新数据库的逻辑
    fmt.Println("Updating database...")
    // 这里可以编写具体的数据库更新代码
}

0 0 2 * * *

具体解释如下：

  • 第一个字段（秒）为0，表示在每分钟的0秒时触发。
  • 第二个字段（分钟）为0，表示在每小时的0分时触发。
  • 第三个字段（小时）为2，表示在每天的2点时触发。
  • 第四个字段（日期）为*，表示每天都触发，不限制日期。
  • 第五个字段（月份）为*，表示每个月都触发，不限制月份。
  • 第六个字段（星期几）为*，表示每个星期都触发，不限制星期几。

"0 0 2 * * *" 表达的含义是在每天凌晨2点时触发定时任务。根据这个cron表达式配置的定时任务将在每天的凌晨2点执行。
