基础命令

# 基本格式
docker run [options] image [command] [arg..]

# 示例
docker run --name=mynginx nginx:latest

# 示例2
docker run -d -p 80:80 docker/getting-started

# 参数说明
-d: 在后台运行容器
-p: 80:80 将主机的80端口映射到容器的80端口
dokcer/getting-started 使用的镜像

# 上面的命令可以简写为
docker run -dp 80:80 docker/getting-started

Docker file

dockerfile就是执行一系列的docker命令的一个脚本。例如：

# syntax=docker/dockerfile:1
FROM node:12-alpine
RUN apk add --no-cache python2 g++ make
WORKDIR /app
COPY . .
RUN yarn install --production
CMD ["node", "src/index.js"]
EXPOSE 3000
# 启动命令
docker build -t getting-started .
# 开启
docker run -dp 3000:3000 getting-started

Docker update

更新文件内容之后，继续执行命令

docker build -t getting-started .

使用更新代码，开启一个新的容器

docker run -dp 3000:3000 getting-started

但是此时会出现一个端口占用错误。
移除旧的container，使用cli的方式

docker ps

停止容器并移除
使用docker stop命令停止容器

docker stop <the-container-id>

容器停止以后，就可以通过rm命令移除

docker rm <the-container-id>

强制移除
增加force flag，用于强制移除容器

docker rm -f <the-container-id>

启动更新后的容器

docker run -dp 3000:3000 getting-started

Docker share

docker hub分享

分析docker images之前，你必须要有一个docker registry，默认的是docker hub，这也是我们使用的所有镜像的来源地址。

推送的时候需要加id，不能直接推。
例如

docker push docker/getting-started

登录docker hub

docker login -u docker-id

使用docker tag命令给getting-started重新命名

docker tag getting-started docker-id/getting-started

推送

docker push docker-id/getting-started

在这个命令行下，开启最新的推送app

docker run -dp 3000:3000 docker-id/getting-started

推送一个新的tag到仓库中

docker push docker/getting-started:tagname

拷贝分享

拷贝docker images

Docker save -o xxx image

在新的机器或位置上加载

Docker load -I xxx

xxx为分享时的打包镜像名

挂载

假如我们只想简单的存储数据，名称卷是个好方法，因此我们不关心这数据存在哪。
用绑定挂载，我们在主机上控制的挂载点。我们用这种办法去持久化数据，但是这种只是经常被用于提供容器中的额外数据。
当在一个应用中工作时，我们创建一个绑定挂载点用于挂载我们的源代码，可以在容器中让我们看到代码的改变，响应，以及实时变化。

对于基于节点的应用，nodemon是一个好的工具用于监控文件变化并重启应用。

开启一个测试容器

docker run -dp 3000:3000 \
-w /app -v "$(pwd):/app" \
node:12-alpine \
sh -c "yarn install && yarn run dev"

windows下的命令：

docker run -dp 3000:3000 `
-w /app -v "$(pwd):/app" `
node:12-alpine `
sh -c "yarn install && yarn run dev"

说明：
-dp 3000:3000 后台运行，映射端口
-w /app 设置工作目录，或者当前的目录用于命令执行来源
-v “$(pwd):/app” 绑定从容器主机中的当前文件到/app文件夹
node:12-alpine 使用的镜像
sh -c “yarn install && yarn run dev” 执行命令

查看容器日志

docker logs -f <container-id>
