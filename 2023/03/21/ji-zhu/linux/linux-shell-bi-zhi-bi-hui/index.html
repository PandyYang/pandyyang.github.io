Linux Shell

概述

Linux提供的Shell解析器

[root@VM-8-10-centos bin]# cat /etc/shells

/bin/bash
/usr/bin/sh
/usr/bin/bash
/bin/tcsh
/bin/csh

CentOS默认的bash

[root@VM-8-10-centos bin]# echo $SHELL

Shell脚本初步

格式

以 #!/bin/bash 开头，指定解析器。

例如hello world.sh

touch helloworld.sh

vim helloworld.sh

输入

#!/bin/bash
echo "hello world"

执行的两种方式

  1. sh
sh helloworld.sh
  1. chmod +x
./helloworld.sh

路径可以选择相对路径和绝对路径。

区分子shell和父级shell

[pandy@VM-8-10-centos sh]$ vim test.sh

脚本内容
[pandy@VM-8-10-centos sh]$ cat test.sh
#!/bin/bash
A=5
echo $A

在当前shell中打开子shell
[pandy@VM-8-10-centos sh]$ bash test.sh
5

脚本结束，子shell关闭，回到父shell，无法读取到内容
[pandy@VM-8-10-centos sh]$ echo $A

在当前shell中打开子shell
[pandy@VM-8-10-centos sh]$ sh test.sh
5

脚本结束，子shell关闭，回到shell，无法读取到内容
[pandy@VM-8-10-centos sh]$ echo $A

[pandy@VM-8-10-centos sh]$ chmod +x test.sh
[pandy@VM-8-10-centos sh]$ ./test.sh
5
[pandy@VM-8-10-centos sh]$ echo $A

“.”或者source 都会使脚本内容在当前shell里执行，而无需打开子shell
[pandy@VM-8-10-centos sh]$ . test.sh
5
[pandy@VM-8-10-centos sh]$ echo $A
5

开子shell与不开子shell 的区别在于 环境变量的继承，在子shell中设置当前变量，父shell是不可见的。

$HOME、$PWD、$SHELL、$USER、$PATH

查看当前shell中所有变量

sh

自定义变量

  1. 定义

    变量名=变量值，注意，=号前后不能有空格。

  2. 撤销变量

    unset 变量

  3. 声明静态变量

    readonly 变量 不能unset

变量的定义规则

  • 名称由字母、数字、下划线组成，但是不能以数字开头，环境变量名建议大写。

  • 等号两侧不能有空格。

  • 在bash中，变量默认类型都是字符串类型。无法直接进行数值计算。

  • 变量的值如果有空格，需要使用双引号或者单引号扩起来。

实例：

[pandy@VM-8-10-centos sh]$ A=5
[pandy@VM-8-10-centos sh]$ echo $A
5
[pandy@VM-8-10-centos sh]$ A=8
[pandy@VM-8-10-centos sh]$ echo $A
8
[pandy@VM-8-10-centos sh]$ unset A
[pandy@VM-8-10-centos sh]$ echo $A

[pandy@VM-8-10-centos sh]$ readonly B=2
[pandy@VM-8-10-centos sh]$ echo $B
2
[pandy@VM-8-10-centos sh]$ B=9
bash: B: readonly variable
[pandy@VM-8-10-centos sh]$ unset B
bash: unset: B: cannot unset: readonly variable

特殊变量

$n

功能描述，n为数字，$0代表脚本的名称，$1-$9代表第一到第九个参数，十以上的参数，十以上的参数需要用大括号包含。${10}

#!/bin/bash
echo '=========================$n======================'
echo $0
echo $1
echo $2

[pandy@VM-8-10-centos sh]$ chmod +x parameter.sh
[pandy@VM-8-10-centos sh]$ ./parameter.sh pandy tinyfry mynsiho
=========================$n======================
./parameter.sh
pandy
tinyfry

$#

获取所有输入参数的个数

[pandy@VM-8-10-centos sh]$ cat parameter.sh
#!/bin/bash
echo '=========================$n======================'
echo $0
echo $1
echo $2
echo '=========================$#======================'
echo $#

[pandy@VM-8-10-centos sh]$ ./parameter.sh pandy tinyfry mynsiho
=========================$n======================
./parameter.sh
pandy
tinyfry
=========================$#======================
3

$*

代表命令行中的所有参数 将所有参数作为一个整体

$@

代表命令行中的所有参数 但是将每个参数区分对待

#!/bin/bash
echo '=========================$n======================'
echo $0
echo $1
echo $2
echo '=========================$#======================'
echo $#
echo '=========================#*======================'
echo $*
echo '=========================#@======================'
echo $@

[pandy@VM-8-10-centos sh]$ ./parameter.sh pandy tinyfry mynsiho
=========================$n======================
./parameter.sh
pandy
tinyfry
=========================$#======================
3
=========================#*======================
pandy tinyfry mynsiho
=========================#@======================
pandy tinyfry mynsiho

$?

最后一次执行的命令的返回状态，如果是0，证明上一个命令正确执行。

[pandy@VM-8-10-centos sh]$ echo $?
0

运算符

$((运算式)) 或 $[运算式]

计算（2+3） * 4

[pandy@VM-8-10-centos sh]$ S=$[(2+3)*4]
[pandy@VM-8-10-centos sh]$ echo $S
20

条件判断

基本语法

（1）test condition

（2）[ condition ]（注意condition前后要有空格）

注意：条件非空即为true，[ atguigu ]返回true，[ ] 返回false。

常用判断条件

（1）两个整数之间比较

  • -eq 等于（equal）
  • -ne 不等于（not equal）
  • -lt 小于（less than）
  • -le 小于等于（less equal）
  • -gt 大于（greater than）
  • -ge 大于等于（greater equal）

（2）按照文件权限进行判断

  • -r 有读的权限（read）

  • -w 有写的权限（write）

  • -x 有执行的权限（execute）

（3）按照文件类型进行判断

  • -e 文件存在（existence）

  • -f 文件存在并且是一个常规的文件（file）

  • -d 文件存在并且是一个目录（directory）

22大于23

[pandy@VM-8-10-centos sh]$ [ 22 -ge 23 ]
[pandy@VM-8-10-centos sh]$ echo $?
1

helloworld.sh有读写权限

[pandy@VM-8-10-centos sh]$ [ -w helloworld.sh ]
[pandy@VM-8-10-centos sh]$ echo $?
0

helloworld.sh是否存在

[pandy@VM-8-10-centos sh]$ [ -e /home/pandy/sh/helloworld.sh ]
[pandy@VM-8-10-centos sh]$ echo $?
0

多条件判断

多条件判断（&& 表示前一条命令执行成功时，才执行后一条命令，|| 表示上一条命令执行失败后，才执行下一条命令）

[pandy@VM-8-10-centos sh]$ [ pandy ] && echo OK || echo notOK
OK
[pandy@VM-8-10-centos sh]$ [ ] && echo OK || echo notOK
notOK

练习

输入一个名字，如果是pandy，输出干啥啥不行，如果不是，输出，吃饭第一名。

#!/bin/bash

pandy="pandy"
aha="aha"
if [ $1 == $pandy ]
then
    echo "pandy is hansome"
elif [ $1  == $aha ]
then
    echo "everyting is notok"
fi

[pandy@VM-8-10-centos sh]$ sh if.sh pandy
pandy is hansome

case语句

case $变量名 in 
"值1"） 
    如果变量的值等于值1，则执行程序1 
;; 
"值2"） 
    如果变量的值等于值2，则执行程序2 
;; 
    …省略其他分支… 
*） 
    如果变量的值都不是以上的值，则执行此程序 
;; 
esac

（1）case行尾必须为单词“in”，每一个模式匹配必须以右括号“）”结束。

（2）双分号“**;;**”表示命令序列结束，相当于java中的break。

（3）最后的“*）”表示默认模式，相当于java中的default。

示例

输入一个数字，如果是1，输出pandy，如果是2，输出fry，如果是3，输出tiny

#!/bin/bash
case $1 in
"1")
        echo "pandy"
;;

"2")
        echo "fry"
;;
*)
        echo "tiny"
;;
esac

[pandy@VM-8-10-centos sh]$ sh case.sh 1
pandy

for循环

基本语法

for (( 初始值;循环控制条件;变量变化 )) 
do 
    程序 
done

示例


#!/bin/bash
sum=0
for((i=0;i<=100;i++))
do
    sum=$[$sum+$i]
done
echo $sum

[pandy@VM-8-10-centos sh]$ sh for1.sh
5050

基本语法2

for 变量 in 值1 值2 值3… 
do 
    程序 
done

#!/bin/bash

for i in money girl power
do
    echo "pandy love $i"
done

[pandy@VM-8-10-centos sh]$ sh for2.sh
pandy love money
pandy love girl
pandy love power
