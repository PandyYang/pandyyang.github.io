情报SDK使用手册

简介

威胁情报SDK是一种软件开发工具包，用于为应用程序和系统提供实时威胁情报。

该SDK通过与xx的情报数据接口集成，向开发人员提供威胁情报数据，以便用户可以在其应用程序中使用这些数据来保护其用户或设备。

威胁情报SDK包含以下功能：

  • IP地址、域名、文件哈希等的威胁情报查询
  • 支持多种编程语言集成
  • 支持以CLI的方式进行操作或管理
  • 支持以RESTful的方式进行跨平台使用

此SDK致力于帮助开发人员提高应用程序的安全性，并为用户或设备提供更好的保护。

安装前的准备工作

  1. 确保运行SDK的机器为AMD的x86_64架构。
  2. 确保机器可用内存及磁盘拥有足够资源。

初始化

先执行bin目录下的gethash以获取机器hash，随后使用此hash联系相关人员申请该机器的专属证书。

# 切换至bin目录
cd bin

# 执行程序gethash
./gethash

SDK的使用

动态链接库文件

具体表现为.so文件和.h文件，可以在各种语言环境下调用动态链接库文件，并且此种形式提供最好的性能。

在使用之前，必须要先引入动态链接库文件，并设置so文件所在的目录为LD_LIBRARY_PATH环境变量。环境变量的设置请参阅注意事项部分。

Golang调用

参数说明：

configPath：配置文件所处的位置。

URL： 要操作的数据类型，有ip、domain、url、sample四种。

addStrKey： 增加数据时的key，类型为字符串。

addStrValue：增加数据时的value，格式为json，参考情报数据结构体说明的value部分。

modifyStrValue：和addStrValue格式类型一致。

api代码：

// 指定动态链接库文件位置（以实际目录为准）
// #include "../../lib/libsdk.h"
// #cgo LDFLAGS:-L../../lib -lsdk
import "C"  

// 初始化SDK
sdkContext := C.initSDK(C.CString(configPath))
// 设备hash
hash := C.queryHash(sdkContext)
// 证书详情
creditDetails := C.queryCreditDetails(sdkContext)
// 增加
addRes := C.add(sdkContext, C.CString(URL), C.CString(addStrKey), C.CString(addStrValue))
// 修改
modifyRes := C.modify(sdkContext, C.CString(URL), C.CString(addStrKey), C.CString(modifyStrValue))
// 删除
C.delete(sdkContext, C.CString(URL), C.CString(addStrKey))
// 释放资源
C.destroy(sdkContext)
Python调用

在示例代码中，有对Python调用方式的一层封装，直接集成进代码中调用即可。

封装了的SDK类：

class SDK():

    def __init__(self) -> None:
        self.p = None
        self.soObj = None

    def initSDK(self):
        dllabspath = os.path.dirname(os.path.abspath(__file__)) + os.path.sep + '../../lib/libsdk.so'
        configabspath = os.path.dirname(os.path.abspath(__file__)) + os.path.sep + '../../conf/config.properties'
        
        so = ctypes.CDLL(dllabspath)

        so.query.restype = ctypes.c_char_p
        so.add.restype = ctypes.c_char_p
        so.modify.restype = ctypes.c_char_p
        so.queryHash.restype = ctypes.c_char_p
        so.queryCreditDetails.restype = ctypes.c_char_p
        
        # 初始化sdk
        ptr = so.initSDK(ctypes.c_char_p(configabspath.encode("utf-8")))    
        self.p = ctypes.c_void_p(ptr)
        self.soObj = so

    def query(self, type, key):
        res = self.soObj.query(self.p, ctypes.c_char_p(type.encode("utf-8")), ctypes.c_char_p(key.encode("utf-8")))
        return res.decode()
    
    def add(self, type, key, value):
        t = ctypes.c_char_p(type.encode("utf-8"))
        k = ctypes.c_char_p(key.encode("utf-8"))
        v = ctypes.c_char_p(value.encode("utf-8"))
        res = self.soObj.add(self.p, t, k, v)
        return res.decode()

    def modify(self, type, key, value):
        t = ctypes.c_char_p(type.encode("utf-8"))
        k = ctypes.c_char_p(key.encode("utf-8"))
        v = ctypes.c_char_p(value.encode("utf-8"))
        res = self.soObj.modify(self.p, t, k, v)
        return res.decode()

    def delete(self, type, key):
        t = ctypes.c_char_p(type.encode("utf-8"))
        k = ctypes.c_char_p(key.encode("utf-8"))
        self.soObj.delete(self.p, t, k)
    
    def queryHash(self):
        res = self.soObj.queryHash(self.p)
        return res.decode()

    def queryCreditDetails(self):
        res = self.soObj.queryCreditDetails(self.p)
        return res.decode()

    def destroy(self):
        if self.p == 0:
            return
        self.soObj.destroy(self.p)

实例化SDK类进行使用：

t: 情报类型，有ip、domain、url、simple四种类型。

k: 情报数据的key，详情请参阅情报数据结构体说明。

v: 情报数据的value，详情请参阅情报数据结构体说明。

v2: 与v格式相同，详情请参阅情报数据结构体说明。

# 初始化
sdk = SDK()
sdk.initSDK()

# 查询机器hash
queryHashRes = sdk.queryHash()

# 查询证书信息
queryCreditDetailsRes = sdk.queryCreditDetails()

# 增加
addRes = sdk.add("domain", k, v)

# 验证增加
validAddRes = sdk.query("domain", k)

# 修改
modifyRes = sdk.modify("domain", k, v2)

# 删除
sdk.delete("domain", k)

# 关闭sdk
sdk.destroy()

CLI的使用

形式为Linux下的可执行文件。直接运行此文件，即可在CLI中对情报数据进行操作，对机器hash及证书详情进行查询。

执行cli，查看其包含的所有命令及解释说明。

./cli --help

Usage of ./cli:
  -g, --config string                    import config file, can not be null, by default 																						 will read config file in the same directory
  -f, --filePath string                  import offline data package
  -t, --operateType string               the operate type, for example: ip, domain, url                 																				 sample
  -a, --add string                       add multiple data, input the json data
  -d, --delete string                    delete multiple data, input the key separated by 																			     spaces
  -q, --query string                     query multiple data, input the key separated by 									                           spaces
  -m, --modify string                    modify multiple data, input the json data 
      --hash string[="default"]          view the current machine hash, no need value
      --certificate string[="default"]   view certificate details, no need value

参数说明：

**

命令 说明
-g, –config 导入配置文件，如果不写，则默认在conf目录（目前的相对位置）中扫描配置
-f, –filePath 导入加密数据至内存或redis存储
-t, –operateType 要操作的数据类型，有ip、domain、simple、url四种
-a, –add 增加数据，输入格式为key和value组成的json结构体，详情参见情报数据结构体说明
-d, –delete 删除数据，如果要删除多条数据，key之间以空格作为分隔符。
-q, –query 查询数据，如果要查询多条数据，key之间以空格作为分隔符。
-m, –modify 修改数据，输入格式为key和value组成的json结构体，详情参见情报数据结构体说明
–hash 查询设备hash
–certificate 查询证书详情

使用示例

  • 查询设备hash

    ./cli --hash

  • 查询证书详情

    ./cli --certificate

  • 增加数据

    ./cli -t url -a "{\"key\":\"test_url.com\",\"value\":{\"confidence\":90,\"modified\":\"2022-11-26T01:14:36.000Z\",\"observables\":[{\"type\":\"url\",\"value\":\"85c8a27c6ac6ff4b716178eac2f864e5\"}],\"threat_types\":[\"恶意内容网站\"],\"threat_level\":5,\"id\":\"0c90113d6145c780090d2\",\"categories\":[\"url\"],\"revoked\":false}}"

  • 查询数据

    ./cli -t url -q test_url.com

  • 修改数据

    ./cli -t url -a "{\"key\":\"test_url.com\",\"value\":{\"confidence\":90,\"modified\":\"2022-11-26T01:14:36.000Z\",\"observables\":[{\"type\":\"url\",\"value\":\"这是修改后的数据\"}],\"threat_types\":[\"恶意内容网站\"],\"threat_level\":5,\"id\":\"0c90113d6145c780090d2\",\"categories\":[\"url\"],\"revoked\":false}}"

  • 删除数据

    ./cli -t url -d test_url.com

注意事项

  1. 需要将动态链接库文件加载进系统环境变量才能正常使用SDK。具体步骤如下。

    方案1：临时方案。

    假如so文件在/root/goProject/src/xx-SDK/c下, 则在cmd中执行:

    export LD_LIBRARY_PATH="/root/goProject/src/xx-SDK/c"

    当cmd关闭时，环境变量失效，并且无法在其他cmd生效。

    方案2: 永久方案。

    Linux动态库的默认搜索路径是/lib和/usr/lib，我们可以将so文件放到这两个目录中，或者用以下方式指定扫描路径。两种方式都能永久生效。

    编辑etc下的ld.so.conf

    vim /etc/ld.so.conf

    在其中加入so所在的目录。例如：/root/goProject/src/xx-SDK/lib。

    include ld.so.conf.d/*.conf
/root/goProject/src/xx-SDK/lib

    退出保存之后，使配置生效。

    ldconfig

  2. 运行环境要求。

    SDK基于Golang开发。具体版本信息为：

    go version go1.19.3 linux/amd64

    如果您需要在Golang中集成SDK，请确保您当前版本支持CGO，推荐使用1.19及以上版本。

    Python示例程序中的Python基于版本3.7.4，如果您需要运行此示例，请确保安装Python3版本。

    系统开发及测试的GCC版本如下，仅供参考。

    gcc version 4.8.5 20150623 (Red Hat 4.8.5-44) (GCC)
