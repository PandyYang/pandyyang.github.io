使用Python2.7进行日期的格式化
Python
技术
发布日期:   2023-02-15
作者:   Pandy
文章字数:   291
阅读时长:   1 分

昨天需要将数据入库ES，本来之前是经过kafka，再经入库程序消费之后，处理入库。但是搭建的测试环境不好迁移入库程序，所以自己写了一个脚本进行处理。

最后发现es的索引日期类型为UTC的字符串格式，而原数据为int类型。所以需要有一个转换。t的类型为int类的秒值，例如int(time.time())，它表示的是自从1970年1月1日00:00:00开始按秒计算的偏移量。

def convert_time_to_date(t):
    milliseconds = int(round(t * 1000))
    date = time.strftime('%Y-%m-%dT%H:%M:%S', time.localtime(milliseconds / 1000))
    date_with_milliseconds = "%s.%03dZ" % (date, milliseconds % 1000)
    return date_with_milliseconds

此段程序保留个三位毫秒时间戳。因为es的索引date结构为：

"modified": &#123;
"format": "YYYY-MM-dd'T'HH:mm:ss.SSS'Z'",
"type": "date"
&#125;

Python的日期是无法精确到毫秒的。所以需要手动进行转换。

.%03d % (milliseconds % 1000)

毫秒除以1000，为取毫秒的三位浮点数值。

localtime则根据本地时间进行计算，strftime则主要是格式化时间，和Java的formatter差不多。

测试用例：

print convert_time_to_date(int(time.time()))

output: 

2023-02-14T22:02:45.000Z
