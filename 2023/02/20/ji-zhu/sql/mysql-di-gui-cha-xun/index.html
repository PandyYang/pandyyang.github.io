MySQL递归查询

MySQL递归查询适用于存在父子级关系的查询操作，一般复杂的关系组织，建议在表设计时进行解耦，将复杂的关系存储于第三张表中，之后在Service层进行处理。

MySQL的递归查询之前最多是用存储过程进行处理，但是存储过程编写复杂，而且比较重，虽然性能可以但是效果差强人意。MySQL从8开始，支持在sql层面进行递归查询。

-- mysql递归查询
SELECT
    id 
FROM
    (
    SELECT
        t1.*,
        IF-- 查询是否是上下级 逻辑为：判断当前不变变化的节点，在不在不断合并的节点中
        ( FIND_IN_SET( t1.parent_id, @parent_ids ) > 0, @parent_ids := CONCAT( @parent_ids, ',', id ), '0' ) AS ischild 
    FROM
        ( SELECT * FROM sys_department ) t1,-- department
        ( SELECT @parent_ids := '1' ) t2 -- 变量初始化
        
    ) t3 
WHERE
    t3.ischild != '0' UNION-- 自身
SELECT
    id 
FROM
    sys_department 
WHERE
    id = 1
