用户及权限

添加、修改、删除用户

添加账号

useradd 选项 用户名
  • 选项
    • -c commant指定一段注释
    • -d 目录，指定用户主目录，如果不存在，则同时使用-m选项，可以相创建主目录。
    • -g 用户组，指定用户所处的用户组。
    • -G 用户组，指定用户所属的附加组。
    • -s Shell文件，指定用户的登录Shell
    • -u 用户号，指定用户的用户号，如果有-o选项，可以重复使用其他用户的标识号。
  • 用户名
    • 指定新账号的登录名

示例：

useradd -s /bin/sh -g group -G root,adm -d /home/pandy -m pandy

创建一个用户pandy，指定其登陆shell为/bin/sh，为其创建组group, 同时设置附加组root和adm，而group是主组，指定其目录位于/home/pandy，如果此目录不存在，则创建。

删除账号

删除账户就是要将/etc/passwd等系统文件中的该用户记录删除，必要时还删除用户的主目录。

userdel 选项 用户名

  • 选项

    • -r 把用户的主目录一起删除
    userdel -r pandy

    删除pandy，并连带其主目录。同时删除其在系统文件中的记录（/etc/passwd, /etc/shadow/, /etc/group)

修改账号

修改用户账号就是根据实际情况更改用户的有关属性，如用户号、主目录、用户组、登陆shell等。

修改已有的用户信息使用usermod命令。

usermod 选项 用户名

选项和添加用户的选项相同。

有的系统可以使用-l，表示将原来的用户名改为新的用户名。

usermod -s /bin/ksh -d /home/z -g developer sam

此命令将用户sam的登陆Shell修改为ksh，主目录改为/home/z, 用户组改为developer。

用户口令的管理

用户账号刚创建时没有口令，但是被系统锁定无法使用，必须为其指定口令后才可以使用，即使是指空口令。

passwd 选项 用户名
  • 选项
    • -l 锁定口令，即禁用账号。
    • -u 口令解锁
    • -d 使账号无口令
    • -f 强迫用户下次登陆时修改口令

用户组的管理

每个用户都有一个用户组，系统可以对一个用户组中的用户进行集中管理，linux下的用户属于与它同名的用户组，这个用户组在创建用户时同时创建。

用户组的管理涉及到用户组的添加，删除和修改。组的增加、删除和修改实际上就是对/etc/group文件的更新。

  1. 增加一个新的用户组使用groupadd命令，其格式如下。

    groupadd 选项 用户组
    • 选项
      • -g GID 指定新用户组的标识号
      • -o 一般与-g选项同时使用，表示新用户组的GID可以与系统已有用户组的GID相同。

    示例：

    groupadd group1

    此命令向系统中增加了一个新组group1，新组的组标识号实在当前已有的最大标识号的基础上加1.

    groupadd -g 101 group2

    此命令向系统中增加一个新组group2，同时指定新组的组标识号是101.

  2. 删除用户组

    groupdel 用户组

    例如：

    groupdel group1

    此命令从系统中删除组group1

  3. 修改用户组的属性，使用groupmod命令，其语法如下：

    groupmod 选项 用户组

    常用的选项有：

    • -g GID 为用户组指定新的组标识号。
    • -o 与-g选项同时使用，用户组的新GID可以与系统已有用户组的GID相同。
    • -n新用户组，将用户组的名字改为新名字。

    示例：

    groupmod -g 102 group2

    此命令将组group2的组标识号修改为102

    groupmod -g 10000 -n group3 group2

    此命令将组group2的标识号修改为10000，组名修改为group3.

  4. 如果一个用户同时属于多个用户组，用户可以在用户组之间切换，以便具有其他用户组的权限。

    用户登陆后，可以使用newgrp切换到其他用户组，例如。

    newgrp root

    如果root组是当前用户的主组或附加组，将其切换至root。

磁盘

磁盘管理的常用命令为df、du、以及fdisk。

df (disk free) :列出文件系统的整体磁盘使用量。

du (disk used): 列出磁盘空间使用量。

fdisk：用于磁盘分区。

df

此命令用于检查文件系统的磁盘占用情况，可以利用该命令来获取磁盘被占用了多少空间，目前还剩下多少空间等信息。

df [-ahikHTm] [目录或文件名]

选项与参数：

  • -a ：列出所有的文件系统，包括系统特有的 /proc 等文件系统；
  • -k ：以 KBytes 的容量显示各文件系统；
  • -m ：以 MBytes 的容量显示各文件系统；
  • -h ：以人们较易阅读的 GBytes, MBytes, KBytes 等格式自行显示；
  • -H ：以 M=1000K 取代 M=1024K 的进位方式；
  • -T ：显示文件系统类型, 连同该 partition 的 filesystem 名称 (例如 ext3) 也列出；
  • -i ：不用硬盘容量，而以 inode 的数量来显示

du

与 df 命令不同的是 Linux du 命令是对文件和目录磁盘使用的空间的查看。

du [-ahskm] 文件或目录名称
  • -a ：列出所有的文件与目录容量，因为默认仅统计目录底下的文件量而已。
  • -h ：以人们较易读的容量格式 (G/M) 显示；
  • -s ：列出总量而已，而不列出每个各别的目录占用容量；
  • -S ：不包括子目录下的总计，与 -s 有点差别。
  • -k ：以 KBytes 列出容量显示；
  • -m ：以 MBytes 列出容量显示；

  1. 只列出当前目录下的所有文件夹容量。

    du

  2. 将文件的容量和列出

    du -a

  3. 检查根目录底下每个目录所占用的容量

    du -sm /*

文件和目录

文件

Linux系统是一种多用户系统，不同的用户处于不同的地位，拥有不同的权限。

为了保护系统的安全性，Linux对不同的用户访问统一文件的权限做了不同的规定。

  • chown (change owner): 修改所属用户与组。
  • chmod (change mode): 修改用户的权限

用l l 或者 ls -l命令来显示一个文件的属性以及文件所属的用户和组，例如：

[root@VM-8-10-centos home]# ls -l
总用量 16
drwxr-xr-x 5 root       root       4096 9月  22 22:33 go
drwxr-xr-x 3 root       root       4096 8月   4 2022 jdk
drwx------ 6 lighthouse lighthouse 4096 8月  15 2022 lighthouse
drwx------ 4 pandy      pandy      4096 2月  18 14:58 pandy

在结果中，go的第一个属性用d标识，d在linux中代表一个目录文件。

  • d为目录
  • -为文件
  • l 则表示为链接文档
  • b表示为装置文件里面可供存储的接口设备
  • c表示为存储文件里面的串行端口设备，例如键盘鼠标。

接下来的字符中以三个为一组，且均为rwx的三个参数的组合，其中：

  • r 可读
  • w 可写
  • x 可执行

权限位置不会改变，如果没有权限则出现-号。

chgrp 更改文件属组

chgrp [-R] 属性组 文件名
  • -R 递归更改文件属组。

chown 更改文件属主/属组

chown [-R] 属主名 文件名

chown [-R] 属主名：属组名 文件名

chmod：更改文件9个属性

linux文件基本权限有9个，分别是owner、group、others，三种身份各有自己的rwx权限。

  • r 4
  • w 2
  • 1

每种身份的各自三个权限需要累加，例如当前权限为：-rwxrwx– 分数则是：

  • owner rwx = 4 + 2 + 1 = 7
  • group rwx = 4 + 2 + 1 = 7
  • others = — = 0 + 0 + 0 = 7

变更权限的语法：

chmod [-R] xyz 文件或目录

符号类型改变文件权限

以上九个权限分别是：

  • user 用户
  • group 组
  • others 其他

可以用，u g o代表三种身份的权限。

使用+ - =，代表权限的加入、除去、设定。

如果要将文件设置为：-rwxr-xr–, 可以使用chmod u=rwx,g=rx,o=r来设定。
